# PaperSmith 学术润色

使用这个 skill 对论文文本进行学术表达、逻辑结构和风格润色。

## 适用场景

- 用户提供一段论文文本，要求润色。
- 用户要求把文本改得更学术、更自然、更符合期刊写作。
- 用户要求优化摘要、引言、相关工作、方法、结果、讨论或结论。
- 用户要求润色后发送到 PaperSmith。

## 润色原则

1. 保留原文核心意思，不新增未经用户提供的数据、结论或引用。
2. 优先提升清晰度、逻辑衔接、术语一致性和学术语气。
3. 避免过度包装、空泛强化和没有证据支撑的强因果表达。
4. 如果是英文论文，使用自然、克制、可发表的学术英语。
5. 如果涉及 APA 风格，关注表达、引用叙述、标题层级和术语一致性；不要伪造参考文献。

## 工作流程

1. 判断用户是否已经提供文本。
2. 如果用户要求润色 PaperSmith 当前选区，转入 `papersmith-rewrite-selection`。
3. 如果用户提供的是 PaperSmith feedback markdown，转入 `papersmith-revise-from-feedback`。
4. 对普通文本直接润色，并在聊天中输出结果。
5. 如果用户要求同步到 PaperSmith，使用 `papersmith-insert-text` 追加写入。

## 输出方式

- 简单润色：直接给出润色后版本。
- 复杂润色：先给“润色后版本”，再列出少量关键修改说明。
- 不要默认输出逐句解释，除非用户要求。

## 质量检查

输出前快速检查：

- 是否保留了原始论点。
- 是否避免了新增事实。
- 是否减少了口语化表达。
- 是否保持段落内部逻辑连贯。
- 是否符合用户指定语言。
