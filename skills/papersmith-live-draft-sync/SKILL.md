# PaperSmith Live Draft Sync

这个 workflow 负责 PaperSmith 打开后的写作模式：Codex 仍然在聊天框内给出正文，但凡是应该进入论文编辑器的正文内容，也要同步到 PaperSmith。

## 触发条件

使用本 skill 当：

- PaperSmith 已经打开或本轮任务明确处在 PaperSmith 写作流程中。
- 用户要求写正文、给我正文、发给我 introduction、输出或返回论文段落、改正文、润色正文、生成论文段落、生成修订稿、继续写、扩写、压缩、翻译成论文表达或重写某段。
- 用户说“把之前写的 introduction 发给我”“给我正文”“输出上一版段落”等表达时，如果内容属于论文正文，也按正文同步处理。
- 用户没有明确说“不要同步”“只在聊天里给我”。

不要使用本 skill 当：

- 只是解释工具、汇报状态、给命令、做安装调试、讨论设计方案。
- 输出内容不是正文，而是计划、说明、检查结果、错误原因或操作步骤。
- 用户明确要求不要写入 PaperSmith。

## 输出顺序

必须按这个顺序工作：

1. 先在 Codex 聊天框输出正文，并在最终回复的最前面完整显示正文。
2. 然后调用 `insert_papersmith_text`，只把正文传给 PaperSmith。
3. 最后再输出正文之外的说明、修改理由、提醒或下一步。

正文部分也要在 Codex 聊天框内显示；不要只同步到 PaperSmith 而不在聊天框展示。

## 同步范围

- 同步到 PaperSmith 的 `text` 只能是正文、修订稿或论文段落本身。
- 不要把说明、计划、命令、调试日志或元信息同步到 PaperSmith。
- 如果回复里有“正文 + 说明”，只同步正文；说明留在正文之后的聊天文字里。
- 如果有多个候选版本，除非用户要求全部同步，只同步推荐版本，并在说明里简要标明还有其他候选。

## 反馈格式

推荐回复结构：

```text
[正文]

已同步到 PaperSmith，作为新的 Codex draft。

[可选：简短说明、修改理由、后续建议]
```

如果同步工具失败：

```text
[正文]

PaperSmith 同步失败：[简短原因]
```

不要因为同步失败而省略聊天框里的正文。
