# PaperSmith 选区改写

使用这个 skill 读取 PaperSmith 当前选中文本，并生成局部改写版本。

## 工作流程

1. 确认 PaperSmith 编辑器运行在 `http://127.0.0.1:43227/`。
2. 调用 `get_papersmith_selection` 读取当前选区。
3. 如果没有选中文本，提示用户先在 PaperSmith 中选择需要修改的文本。
4. 根据用户要求改写选中文本。
5. 默认在聊天中给出替代表达；如果用户要求发送到 PaperSmith，使用 `papersmith-insert-text` 追加写入。

## 当前能力边界

- 当前 MCP 可以读取 selection，也可以追加插入文本。
- 当前 MCP 不能直接把新文本替换回原选区。
- 因此不要说“已替换选中文本”；应说“已生成选区改写稿”，或在用户要求时“已将改写稿追加到 PaperSmith”。

## 改写原则

- 只处理选中文本，不擅自改写整段或整篇。
- 保持原意和关键术语，除非用户要求重构论点。
- 学术文本优先提升准确性、逻辑衔接、语气和句法清晰度。
- 如果选区太短导致上下文不足，给出谨慎改写，并提示可能需要周边句子判断衔接。

## 输出方式

简短任务可以直接输出：

```text
改写版本：
...
```

如果有多个候选，可以给 2-3 个版本，并说明差异，例如“更正式”“更简洁”“更强论证”。
